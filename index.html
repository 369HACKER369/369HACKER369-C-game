C++ Knowledge Base

Unit #1: Basic Building Blocks of a Computer Program

Von-Neumann Architecture: Shared memory structure for program instructions and data. Controlled by CPU containing ALU, Control Unit, and Registers.

Constants & Variables: Identifiers pointing to memory locations. Use const for immutable values. Variable names must start with a letter or underscore, not numbers.

Data Types: int (integer), float / double (single/double precision floating-point), char (character), bool (boolean).

Operators & Expressions: Arithmetic (+, -, *, /, %), Comparison (==, !=, <, >, <=, >=), and Logical (&&, ||, !).

Inputs/Outputs: std::cin >> captures console inputs; std::cout << sends outputs to the standard stream console.

Debugging & Errors: Syntax errors violate compiler grammatical rules; Logical errors represent correct grammar but incorrect reasoning, causing unexpected runtime outputs.

Unit #2: Decision Structures

Execution Flow: Standard C++ executes sequentially unless a branching decision modifies control flow based on boolean expressions.

Single Selection (if): Evaluates a body of statements ONLY if the expression evaluates to non-zero/true.

Double Selection (if/else): Branches to the if path if the condition holds true, or the else path if it evaluates to false.

Multiple Selection (switch): Matches a discrete integer/character expression onto multiple label case branches. Execution falls through automatically unless interrupted by a break.

Conditional Operator (?:): Ternary syntax compacts selection choices: condition ? expression_if_true : expression_if_false;.

Nested Decisions: Positioning decision structures within other decisions allows evaluation of hierarchically dependent criteria.

Unit #3: Loops

Repetitive Statements: Enables executing statement blocks repeatedly until designated conditional criteria alter.

Counter Loops (for): Used when iterations can be predetermined. Syntax: for(initialization; condition; increment/decrement).

Conditional Pre-Test Loops (while): Checks condition initially. If true, iterates body; if false on onset, executes 0 times.

Conditional Post-Test Loops (do/while): Executes body once first, then checks condition. Guaranteed to run at least 1 time.

Loop Control & Nesting: break terminates and exits the nearest loop; continue skips remaining statements in current iteration, advancing to next check. Nesting runs inner loops fully for every single iteration of the outer loops.
